ğŸ” AUTHENTICATION ROUTES (prefix: /api/auth)

1. POST /api/auth/login
   - Login with Firebase ID token
   - Get JWT cookie
   
   Body:
   {
     "idToken": "firebase-id-token-here"
   }
   
   Response:
   {
     "success": true,
     "message": "Login successful",
     "user": { "uid": "...", "email": "..." }
   }

2. POST /api/auth/logout
   - Logout and clear JWT cookie
   
   Response:
   {
     "success": true,
     "message": "Logout successful"
   }

3. GET /api/auth/verify
   - Verify if user is authenticated
   - Checks JWT cookie
   
   Response:
   {
     "success": true,
     "authenticated": true,
     "user": { "uid": "...", "email": "..." }
   }

ğŸ” COMMON ISSUES

1. "Route not found" error:
   - Check if you're using correct URL
   - All routes must start with /api/
   - Example: /api/items NOT /items

2. CORS error:
   - Make sure FRONTEND_URL in backend/.env matches your frontend URL
   - Local: http://localhost:3000
   - Production: https://your-netlify-app.netlify.app

3. Authentication error:
   - Make sure you're logged in
   - JWT cookie must be present
   - Cookie is HTTP-only (can't access via JavaScript)
